Команды для создания и управления структурой объектов базы данных.

![[Pasted image 20250810194405.png]]

1. **CREATE TABLE projects** - основная команда, которая создаёт новую таблицу с именем *projects*.
2. **id SERIAL PRIMARY KEY**
	1. **`id`**: Это название первой колонки.
	2. **`SERIAL`**: Это специальный тип данных в PostgreSQL. Он автоматически генерирует уникальные, возрастающие числа для каждой новой строки. По сути, это счётчик.
	3. **`PRIMARY KEY`**: Это ограничение, которое делает колонку **первичным ключом**. Первичный ключ должен быть уникальным и не может содержать `NULL` значения. Он однозначно идентифицирует каждую запись в таблице.
3. **`VARCHAR(100)`**: Это тип данных для хранения строк переменной длины. Число в скобках (`100`) указывает максимальное количество символов, которое можно хранить.
4. **`NOT NULL`**: Это ограничение, которое запрещает оставлять эту колонку пустой. То есть, при добавлении новой записи, поле `name` должно быть обязательно заполнено.
5. **`DATE`**: Это тип данных для хранения даты без времени.



С внешним ключом

![[Pasted image 20250810195145.png]]

- **`FOREIGN KEY (author_id)`**: Указывает, что столбец `author_id` в нашей таблице (`books`) будет внешним ключом.
- **`REFERENCES authors (id)`**: Определяет, что этот ключ ссылается на таблицу `authors`, а конкретно — на её столбец `id`
